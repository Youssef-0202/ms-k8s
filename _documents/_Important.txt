II-  API Gateway usinf Spring Cloud Gateway MVC :  (to search <Functional endpoints models !!>):

Spring Cloud Gateway originally was reactive only, based on WebFlux and Netty.
Not all applications use reactive programming. Some still use the traditional, blocking Spring MVC.
So, the Spring team decided to offer two gateway implementations:
One reactive for apps built with WebFlux.
One blocking (MVC) for apps built with Spring MVC.
This allows developers to choose the gateway that best matches their app’s architecture.

IV- add Circuit Breaker using Resilience4J :
  1- configuration :
      resilience4j:
  circuitbreaker:
    configs:
      default:
        registerHealthIndicator: true // Active un endpoint Spring Boot Actuator de santé (/actuator/health) pour le Circuit Breaker.
        slidingWindowType: COUNT_BASED
        slidingWindowSize: 10 // Nombre d’appels à surveiller dans la fenêtre glissante.
        failureRateThreshold: 50
        waitDurationInOpenState: 5s
        permittedNumberOfCallsInHalfOpenState: 3
        automaticTransitionFromOpenToHalfOpenEnabled: true
        minimumNumberOfCalls: 5

  explication :
  1- les 10 derniers appels seront pris en compte pour évaluer les échecs.
  2- 5 échouent → 50% d’échec → dépasse le failureRateThreshold.
  3- Le circuit passe à OPEN.
  4- Attente de 5s → passe à HALF_OPEN.
  5- 3 appels test sont faits :
      ✅ Si les 3 réussissent → retourne à CLOSED.
      ❌ Si un échoue → retourne à OPEN.
